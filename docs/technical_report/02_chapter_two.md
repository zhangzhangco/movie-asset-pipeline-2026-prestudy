# 第二章 总体设计方案

## 2.1 指导思想与设计原则

### 2.1.1 指导思想 (Guiding Thoughts)
本项目的建设深入贯彻落实国家文化数字化战略，核心在于推动电影数字资产管理模式的根本性变革——即从传统的**“文件管理” (Managing Files)** 向 **“能力管理” (Managing Capabilities)** 转型。

在 2025-2026 年生成式 AI 成为核心生产力的时代背景下，不仅要建设一个静态的资产库，更要构建一个具备**“资产生成力”**的**“能力基础设施”**。系统通过**资产路由管道 (Asset Routing Pipeline, ARP)**，将单帧图像等静态数据智能转化为三维可计算资产，支持从“死数据”到“活世界”的智能反演与确定性重构。

### 2.1.2 设计原则
1.  **统筹规划，标准先行**：
    坚持顶层设计与底层逻辑相统一。严格遵循 GB/T 36369 等国家标准，同时参照 SMPTE、MovieLabs 2030 等国际先进技术规范。建立“一套数据、一套标准、全域流通”的标准化作业体系，避免盲目建设和重复投入，确保系统建成即合规，入库即互通。

2.  **安全可控，底线思维**：
    将数据安全置于首位。针对电影数字资产高价值、高敏感的特性，确立**“内生安全” (Intrinsic Security)** 的防护理念。在基础设施层，全面适配国产信创环境（芯片、操作系统、数据库）；在数据层，采用密码学技术实现资产的防篡改与全链路溯源，坚决守住不发生系统性数据风险的底线。

3.  **技术引领，适度超前**：
    立足当前，着眼未来。针对 AIGC 技术迭代快、颠覆性强的特点，在技术选型上保持**“代际优势”**。坚决摒弃即将被淘汰的传统摄影测量和手工建模路线，大胆采用辐射场（Radiance Fields）与生成式 AI 结合的混合架构，确保平台建成后在未来 5-10 年内依然保持国际领先水平。

4.  **需求牵引，应用为王**：
    坚持“以用促建，建用并举”。不搞空中楼阁式的技术堆砌，紧紧围绕电影制作、游戏开发、文旅体验的一线真实需求。将“资产复用率”和“生产效能提升”作为检验平台建设成效的根本标准。

## 2.2 系统总体架构设计

本系统采用**“管道为核、AI 为基、技能为链”**的新型架构体系。
打破传统的烟囱式系统，以**资产路由管道 (ARP)** 为中央枢纽，连接下层的基础设施与上层的业务应用。架构确立了**“静态 -> 可计算 -> 交互”**的三阶资产演进逻辑，确保资产在生命周期中不断增值。

### 2.2.1 基础设施层 (Infrastructure Layer)
作为系统的物理基石，本层重点解决**“算力自主”**问题。
*   **计算集群**：部署国产异构计算节点，适配海光/华为等国产 GPU，提供满足 3DGS 实时渲染与 NeRF 高负荷训练所需的算力支撑。
*   **存储网络**：构建高性能分布式对象存储系统，支持 PB 级非结构化影像数据的高吞吐读写。
*   **网络通信**：依托国家文化专网，构建高带宽、低延时的内部数据传输通道，确保 DIT 现场数据与云端智算中心的实时同步。

### 2.2.2 数据资源层 (Data Resource Layer) - 资产三阶演进
基于生成式 AI 的特性，本层将资产重构为三个层级：
*   **一级：静态资产 (Static Assets)**：存储未经压缩的 RAW/EXR 母版及扫描数据。这是资产的“物理底片”，作为永久存储与确权的基准。
*   **二级：可计算资产 (Computable Assets)**：基于 3DGS/NeRF 的隐式/半隐式表示。它们不是死文件，而是具备“再生成能力”的参数集，能够根据需求实时推理出任意视角的图像。
*   **三级：交互资产 (Interactive Assets)**：经过 3DGS-SDF 转换后的显式网格（Mesh）与物理绑定（Rig）。具备完整的物理属性与行为逻辑，可直接进入游戏引擎或虚拟拍摄系统进行实时交互。

### 2.2.3 平台支撑层 (Platform Support Layer) - 资产路由管道 (ARP)
本层核心为 **资产路由管道 (Asset Routing Pipeline, ARP)**，它是系统的“中央处理器”：
*   **路由分发机制**：通过 MCP (Model Context Protocol) 协议，自动分析输入资产的特征，将其“路由”至最适合的 AI 技能节点（如 2D转3D、纹理生成、自动绑定）。
*   **智能解构引擎**：集成 3DGS-SDF 混合算法，负责“可计算资产”与“交互资产”之间的确定性转化。
*   **确权服务总线**：集成国密算法与区块链，确保每一次“路由”和“生成”操作都有据可查，实现全链路的版权固化。

### 2.2.4 业务应用层 (Application Layer)
作为系统的对外窗口，提供各类场景化服务：
*   **资产检索与交易子系统**：提供基于语义的自然语言检索及资产授权交易服务。
*   **虚拟制作支撑子系统**：直接对接 LED 虚拟摄影棚，提供实时资产渲染流。
*   **跨界分发接口**：提供标准 API，向游戏引擎（Unity/UE）及文旅展示终端分发合规资产。

## 2.3 技术路线与选型论证

### 2.3.1 视觉重建路线：3DGS-SDF 实现“确定性呈现”
在“可计算资产”向“交互资产”的转化中，电影工业对**“确定性呈现” (Deterministic Presentation)** 有着严苛要求。
*   **否定纯 NeRF**：虽然神经辐射场效果好，但其“黑盒”特性导致结果具有随机性，无法满足电影对光影精确控制的工业标准（UV 不可展、无法手工修型）。
*   **确立 3DGS-SDF 混合架构**：
    *   **确定性几何**：引入 Signed Distance Fields (SDF) 强约束，确保生成的 Mesh 拓扑结构稳定、可预测，满足工业级修模需求。
    *   **高保真材质**：利用 3D Gaussian Splats (3DGS) 捕获各向异性光照，保留真实世界的微观纹理细节。
    *   **结论**：该路线是目前唯一能同时实现 AI 自动化生成与电影级**确定性控制**的工程最优解。

### 2.3.2 资产封装格式：USD 标准的国产化扩展
虽然 Pixar 推出的 **USD (Universal Scene Description)** 已成为事实上的工业标准，但直接使用存在“标准黑箱”风险。
本方案的技术路线是：**“拥抱标准，但不依赖实现”**。
*   **兼容性**：在数据结构上完全兼容 USD 核心规范，确保资产能被主流 DCC 软件读取。
*   **扩展性**：利用 USD 的 Schema 扩展机制，植入 GB/T 36369 定义的国产元数据字段（如 `<cn_doi>`, `<copyright_hash>`）。
*   **自主性**：自主研发 USD 解析与合成算子，不依赖闭源的黑箱库，确保核心数据流转环节的透明可控。

### 2.3.3 确权机制：区块链与数字水印的结合方案
针对资产流通过程中的侵权风险，本方案采用**“明暗结合”**的确权技术路线。
*   **明文确权（溯源）**：通过区块链存证，将 DOI 与权利人信息上链，每一个操作（下载、修改、授权）都在链上留痕，形成不可篡改的证据链。
*   **暗文确权（取证）**：在资产的几何拓扑和纹理频域中嵌入不可见的**鲁棒数字水印**。即使资产被格式转换、重拓扑或遭受 AIGC 风格化攻击，水印依然能够提取，为司法取证提供核心技术支撑。
