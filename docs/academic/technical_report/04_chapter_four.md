# 第四章 项目实施与工程验证

## 4.1 实验环境与国产化适配验证

### 4.1.1 硬件环境构建：全栈国产算力底座
为验证系统的**“自主可控”**能力，项目组搭建了基于纯国产硬件的实验环境，测试系统在脱离西方主流算力生态（NVIDIA CUDA / Intel x86）下的运行表现。
*   **计算节点**：采用 **华为 Atlas 800 推理服务器**（搭载 Ascend 910B 处理器）作为核心算力单元。针对 CUDA 生态的依赖问题，利用 CANN（Compute Architecture for Neural Networks）异构计算架构进行了算子迁移，成功将 3DGS 的核心渲染算子移植到昇腾 NPU 上运行。
*   **通用处理**：采用 **海光 7000 系列 CPU**（x86 架构兼容国产芯），承载逻辑控制与微服务调度。
*   **操作系统**：全系部署 **麒麟 V10 (KylinOS)** 服务器版，经 7x24 小时高负载测试，系统运行稳定，无崩溃记录。

### 4.1.2 软件栈依赖分析与国产化替代
项目组对系统涉及的 127 个第三方依赖库进行了地毯式排查，根据安全风险等级实施了分类替代策略。
*   **高风险组件（闭源/商用）**：如 NVLink互联协议、Windows图形栈。**替代方案**：采用 OpenEuler 社区开源图形驱动及 HCCL 集合通信库。
*   **中风险组件（开源但在美管控清单）**：如 TensorFlow（早期版本）。**替代方案**：全面迁移至 **百度飞桨 (PaddlePaddle)** 或 **华为 MindSpore** 深度学习框架。
*   **验证结论**：通过代码重构与编译优化，本系统核心代码的国产化率已达到 **100%**，彻底消除了“断供”风险。

## 4.2 典型场景验证：马车巡游长镜头实验

### 4.2.1 实验背景与数据选取
为检验系统在“极限工况”下的生存能力，实验特别选取了某头部古装电影项目中公认最难处理的素材——**“马车街头巡游”**长镜头（4K分辨率，时长 24秒，共 576 帧）。
*   **难点一**：**严重遮挡**。复杂的车轮辐条高速旋转，反复遮挡后方建筑，且存在严重的运动模糊。
*   **难点二**：**高反光材质**。马车上的铜制装饰、丝绸帷幔属于典型的非朗伯体（Non-Lambertian），传统摄影测量几乎无法重建。
*   **难点三**：**动态背景**。围观人群与马车产生复杂的交互运动，破坏了 MVS 算法的静态假设。

### 4.2.2 关键难点攻克：从“视觉汤”到“机械体”
在实验中，智能解构引擎展现出了强大的鲁棒性。
1.  **初始解算**：Mask-RCNN 成功识别并剔除了 92% 的动态人群干扰。
2.  **辐射场优化**：3DGS 利用 3阶球谐函数，完美复现了铜饰在不同角度下的镜面反射光泽，消除了传统建模中常见的“烤死高光”现象。
3.  **结构推断（亮点）**：针对旋转的车轮，GNN 结构推断模组没有将其视为一团模糊的网格，而是准确识别其为圆柱体拓扑，并自动推断出旋转轴心（Pivot Point）。
4.  **最终输出**：系统输出的 USD 资产中，车轮不仅模型完整，且被自动标记为 `Joint: Revolute`（旋转副）。这意味着进入游戏引擎后，该车轮可以直接响应物理引擎驱动，无需人工重新绑定。

### 4.2.3 全流程自动化处理演示
经实测，从数据“拖入”系统到最终资产“产出”，全过程仅耗时 **28分钟**。期间无需任何人工打点、清理噪点或修复破面。这种“无人值守”的作业能力，标志着电影资产生产正式迈入了**“黑灯工厂”**时代。

## 4.3 效能评估与对比分析

### 4.3.1 生产效率对比（人工 vs 智能）
项目组邀请了 3 位资深电影数字资产师（平均从业 10年以上），针对同一组素材进行传统手工建模，并与本系统进行背对背测试。

| 评估指标 | 传统手工建模 (Manual) | 智能摄取管线 (Intelligent Ingestion) | 效能提升 (Efficiency Gain) |
| :--- | :--- | :--- | :--- |
| **平均耗时** | 32 工时 (4天) | 0.5 工时 (28分钟) | **提升 64倍 (↑)** |
| **人力投入** | 1名高级模型师 + 1名材质师 | 0 人 (全自动) | **人力成本趋近于零** |
| **还原精度** | 依赖美术主观判断 (易失真) | 毫米级物理还原 (客观真实) | **精度标准化** |
| **可编辑性** | 优 (原生拓扑) | 良 (SDF重拓扑，满足 90% 需求) | **工程可用** |

### 4.3.2 边际成本分析
*   **传统模式**：属于**劳动密集型**。每增加一个资产，成本呈线性增长（需支付昂贵的人力费用，约 3000-5000元/资产）。
*   **智能模式**：属于**算力密集型**。前期研发成本虽高，但一旦模型部署，每增加一个资产的边际成本仅为电费与算力摊销（约 5-10元/资产）。
*   **结论**：随着资产库规模的扩大，智能模式的成本优势将呈现指数级放大。这使得构建“百万级”国家数字资产库在经济上成为可能。

### 4.3.3 资产复用率与标准化程度评估
在确权与流通环节，传统模式生产的资产由于命名混乱、元数据缺失（仅有文件名），复用率不足 5%。
而本系统产出的资产，通过内嵌 DOI 和 GB/T 36369 元数据，在跨系统（Maya -> Unreal Engine）导入测试中，元数据丢失率为 **0%**。所有权利信息、标签信息均能被目标软件准确解析。这种高度的**标准化 (Standardization)**，是实现全行业“一次生产，跨界复用”的技术基石。
