# 风格与反例指南 (Style & Anti-Patterns Guide)

## 1. 核心反模式：AI 味词汇表 (Banned Vocabulary)
以下词汇触发“图灵测试失利”报警：

| 词汇/句式 | 原因 | 修正方案 |
| :--- | :--- | :--- |
| **"旨在" (aims to)** | 虚词，缺乏行动力 | 直接说"做了什么"，而非"想做什么" |
| **"并非" (is not merely)** | 典型的 AI 辩证法 | 肯定句陈述核心价值，如"这超越了..." |
| **"首先、其次、最后"** | 廉价的逻辑路标，切断文气 | 使用**逻辑推流**，用隐式因果连接 |
| **"并非...而是..."** | 生硬的二元对立 | 直接陈述后者，或用"摒弃了 A，确立了 B" |
| **"值得注意的是"** | 废话 | 删掉。如果真的很重要，直接写内容。 |
| **"具体而言"** | 逻辑切割 | 删掉。直接开始描述细节。 |
| **"标志着..."** | 教科书式总结 | 改为"这彻底改变了..."或直接陈述结果 |
| **"旧范式...新范式"** | 套路化 | 直接陈述方案的必然性 |
| **形容词堆砌** | "核心定义"、"关键步骤" | 删掉形容词。定义就是定义。 |
| **段尾总结** | 每个自然段最后总结一句 | 删掉。段落结束就是结束。 |

## 2. 句子构造示例

### 2.1 线性逻辑引导 (The Golden Thread)
*   **Bad (AI)**: "该系统具有三个优点：一是快，二是稳，三是强。此外，它还支持..."
*   **Good (Human)**: "传统系统因数据同构性差，导致跨部门调用受阻。**为此**，我们重构了数据底座，**从而**实现了秒级响应。这一能力的突破，**使得**跨界应用成为可能。"
*   **Rule**: 句子之间必须有隐形的 "因果链"，带着读者的脑子走。不需要 "Firstly"，逻辑自然流淌。

### 2.2 节奏控制 (Rhythm)
*   **Pattern**: [长句铺垫背景] -> [中句阐述逻辑] -> [短句一锤定音]。
*   **Example**: "面对海量非结构化影像数据确权难、溯源难的行业共性痛点（长），本平台引入了智能语义拆解管线（中），**彻底解决了资产的法律身份问题（短）。**"

### 2.3 技术炫技 (Academic Flexing)
*   ❌ (Plain): "我们用了 AI 来做 3D 重建。"
*   ✅ (Flexing): "系统深度融合了 **大语言模型 (LLM) 的语义理解能力** 与 **神经辐射场 (NeRF) 的隐式表达机制**，并引入 **扩散模型 (Diffusion Priors)** 作为几何生成的归纳偏置..."

## 3. 完整段落重写示例

> **[Background]**: 电影工业长期受困于“资产孤岛”效应，单部电影的数字资产在上映后即沦为沉没成本。
>
> **[Problem -> Solution]**: 这种高投入、低复用的模式，在“数据要素”时代已不再可持续。因此，构建一个国家级的、具备智能拆解能力的资产平台，不再是单纯的技术升级，而是**行业生产关系的重塑**。
>
> **[Technical Verification]**: 通过 Phase 8 原型的实测，我们证明了：**机器可以读懂电影**。这一能力的解封，意味着存量巨大的胶片库将转变为取之不尽的结构化资产金矿。
